## 🎓 Glossario Termini

| Termine | Spiegazione |
|---------|-------------|
| **identificativo** | Struttura WSDL complessa con tipo (P=PinCode) e valore (Base64 cifrato) |
| **parent_child** | Pattern di naming per elementi annidati: `parent_child` |
| **EspandiIdentificativo** | Metodo che trasforma `identificativo` semplice in struttura annidata |
| **SoapServiceInterceptor** | Classe di logging SOAP con file e console output |
| **Logging SOAP** | Registrazione di richieste/risposte SOAP con timestamp e XML |
| **SoapUI** | Tool esterno per testare servizi SOAP |
| **Backward Compatible** | Modifiche che non rompono il codice esistente |
| **KV Format** | Formato chiave=valore (Key=Value) |
| **WSDL** | Web Services Description Language - specifica del servizio |

## 🎁 Cheat Sheet

### Per Usare CreateAuth
```csharp
var client = new PrescriptionClient(config);
string input = "userId=...;identificativo_tipo=P;identificativo_valore=...;contesto=RICETTA-DEM";
string output = client.Chiama(DigitalPrescriptionService.CreateAuth, input);
```

### Per Vedere il SOAP
```
📁 bin/Debug/Logs/SOAP/
   01_Request_ChiamaServizio_*.xml
   02_Response_ChiamaServizio_*.xml
```

### Per Testare su SoapUI
1. Copia `01_Request_*.xml`
2. Incolla in SoapUI
3. Sostituisci valori sensibili
4. Invia
